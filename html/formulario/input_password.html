- Colocar senhas de nameira segura
- Esconde o que está sendo difitado no campo
- O estilo da apresentação do camo, depende do user agent


#Atributos
- minlength / maxlength - o número mńimo/máximo de caracteres
- size - o número aceitável de caracteres para o campo
* pattern
- expressão regular para validar o que está sendo digitado
- altamente recomendado o uso de um padrão de segurança alta
- exemplo:
senha contendo caracteres hexadecimais com o limite de no mínimo 4 eno máximo 8 caracteres - pattern="[0-9a-fA-F]{4,8}"

# placeholder
- Mostra um exemplo de texto a ser difitado no campo

# readony/disabled - atributo booleano indicando se o campo está habilitado ou não

# required
- campo obrigatório

# inputmode
- poderá alterar o uso do teclado em smartphones
- exemplo: queremos que o cliente só adicione números - inputmode="numeric"

# autocomplete
- on: permite a sugestão de: new-password ou current-password
- off: desabilita a opção de autocompletar
- new-password - o navegador poderá sugerir uma nova senha
