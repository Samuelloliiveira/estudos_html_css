- Espera que o usuário digete um email
- Irá validar se o valor digitado é um email
# Atributos
- placeholder
- readonly / disabled
- valur
- required
* multiple
- O campo irá receber 1 ou mais emails, separado por vírgulas
* minlenght / maxlength
- O mínimo/ máximo valor que o campo irá conter
* size
- valor númerico identificando quantos caracteres o campo deverá conter
* pattern
- uso de expressão regular para validar o campo - exemplo: pattern=".+@rocketseat\.com\.br"
* list
- o id de ua tag datalist que está no mesmo documento
- datalist irá conter uma lista de valores pré definidos
- só apresenta valores compatíveis com o campo
